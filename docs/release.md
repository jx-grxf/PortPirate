# Release Runbook

MacDev preview releases are built on GitHub Actions from tags.

## Release Shape

Release notes should stay useful for people deciding whether to download the app:

- Highlights: user-visible changes and why they matter.
- Fixed: bug fixes with enough technical context to build trust.
- Improved: performance, UI, packaging, and workflow improvements.
- Pull requests: link the PRs included in the release when the release is cut from merged PRs.
- Known limitations: preview limitations such as signing and notarization status.

## Local Preflight

```bash
swift test
./script/build_and_run.sh build-only
codesign --verify --deep --strict dist/MacDev.app
MACDEV_VERSION=0.1.4 ./script/package_dmg.sh
hdiutil imageinfo dist/MacDev-0.1.4.dmg >/dev/null
```

## Tag Signing

MacDev releases should use SSH-signed annotated tags. Before tagging, verify local signing configuration:

```bash
git config --get gpg.format
git config --get user.signingkey
```

`gpg.format` should be `ssh`. If it is configured correctly, `git tag -s` creates an SSH-signed tag.

## Publish

```bash
git tag -s v0.1.4 -m "release v0.1.4"
git push origin v0.1.4
```

The `Release` workflow creates or updates the GitHub Release, uploads `MacDev-<version>.dmg`, and keeps the source-code archives generated by GitHub available on the release page.

After the release asset is published, the workflow posts to the `WEBSITE_DEPLOY_HOOK_URL` secret. That Vercel deploy hook rebuilds `johannesgrof.me`, where MacDev release metadata is fetched from GitHub during the Astro build.

## Distribution Notes

Current preview builds are ad-hoc signed for bundle integrity. Public end-user distribution should move to Developer ID signing, hardened runtime, notarization, and stapling before the app is marketed outside GitHub preview releases.

## Manual Verification Checklist

- GitHub Actions `CI` passes on the release commit.
- GitHub Actions `Release` passes for the release tag.
- The DMG asset is attached to the GitHub Release.
- The website deploy hook step finishes successfully.
- The DMG opens with `hdiutil imageinfo`.
- The app bundle verifies with `codesign --verify --deep --strict`.
- README download links point at the latest release.
