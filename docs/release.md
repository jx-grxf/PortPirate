# Release Runbook

PortPirate preview releases are built on GitHub Actions from tags.

## Release Shape

Release notes should stay useful for people deciding whether to download the app:

- Highlights: user-visible changes and why they matter.
- Fixed: bug fixes with enough technical context to build trust.
- Improved: performance, UI, packaging, and workflow improvements.
- Pull requests: link the PRs included in the release when the release is cut from merged PRs.
- Known limitations: preview limitations such as signing and notarization status.

## Local Preflight

```bash
swift test
./script/build_and_run.sh build-only
codesign --verify --deep --strict dist/PortPirate.app
PORTPIRATE_VERSION=0.2.2 ./script/package_dmg.sh
hdiutil imageinfo dist/PortPirate-0.2.2.dmg >/dev/null
```

Sparkle assets require signing secrets:

```bash
PORTPIRATE_VERSION=0.2.2 \
PORTPIRATE_SPARKLE_PUBLIC_KEY="<public ed25519 key>" \
PORTPIRATE_SPARKLE_PRIVATE_KEY="<private ed25519 key>" \
./script/create_sparkle_assets.sh
```

Use `PORTPIRATE_UPDATE_CHANNEL=beta` for prerelease appcast entries.

## Tag Signing

PortPirate releases should use SSH-signed annotated tags. Before tagging, verify local signing configuration:

```bash
git config --get gpg.format
git config --get user.signingkey
```

`gpg.format` should be `ssh`. If it is configured correctly, `git tag -s` creates an SSH-signed tag.

## Publish

```bash
git tag -s v0.2.2 -m "release v0.2.2"
git push origin v0.2.2
```

The `Release` workflow creates or updates the GitHub Release, uploads `PortPirate-<version>.dmg`, `PortPirate-<version>.zip`, and `appcast.xml`, and keeps the source-code archives generated by GitHub available on the release page.

After the release asset is published, the workflow posts to the `WEBSITE_DEPLOY_HOOK_URL` secret. That Vercel deploy hook rebuilds `johannesgrof.me`, where PortPirate release metadata is fetched from GitHub during the Astro build.

Required release secrets:

- `WEBSITE_DEPLOY_HOOK_URL`: Vercel deploy hook for `johannesgrof.me`.
- `PORTPIRATE_SPARKLE_PUBLIC_KEY`: EdDSA public key embedded in `Info.plist`.
- `PORTPIRATE_SPARKLE_PRIVATE_KEY`: EdDSA private key passed to `generate_appcast`.

## Distribution Notes

Current preview builds are ad-hoc signed for bundle integrity. Public end-user distribution should move to Developer ID signing, hardened runtime, notarization, and stapling before the app is marketed outside GitHub preview releases.

## Manual Verification Checklist

- GitHub Actions `CI` passes on the release commit.
- GitHub Actions `Release` passes for the release tag.
- The DMG asset is attached to the GitHub Release.
- The Sparkle ZIP and `appcast.xml` assets are attached to the GitHub Release.
- The website deploy hook step finishes successfully.
- The DMG opens with `hdiutil imageinfo`.
- The app bundle verifies with `codesign --verify --deep --strict`.
- `appcast.xml` contains no `sparkle:channel` for stable releases and `sparkle:channel="beta"` for beta prereleases.
- README download links point at the latest release.
